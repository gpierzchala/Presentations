Model

1. Requirements – Capturing requirements for the software design and functionality;

2. Design – creation of the system mock to facilitate the process implementation;

3. Implementation – the mock is verified and the implementation of the system begins according to the mock;

4. Code-and-fix – the implementation is now verified and tested by the team;

5. Final deadline – setting up the software after a pilot and testing done. Regular updating, verification and debugging of the software;
